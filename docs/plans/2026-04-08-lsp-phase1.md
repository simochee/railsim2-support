# LSP Phase 1: Tokenizer + Parser + 構文 Diagnostics 実装計画

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** RailSim2 定義ファイルの構文エラーをエディタ上でリアルタイムに表示する Language Server を構築する

**Architecture:** TypeScript で手書きの tokenizer + Pratt parser を実装し、vscode-languageserver/vscode-languageclient で VS Code と接続。構文レベルの diagnostics（`{}` 不一致、`;` 欠落）はパーサーが、不明キーワード警告は validator が返す。`shared/keywords.ts` をキーワード定義の Single Source of Truth とし、validator がそこを参照する。パーサーは `keywords.ts` に依存せず、純粋に構文構造のみを解析する。

**Tech Stack:** TypeScript (tsc), vscode-languageserver, vscode-languageclient, vitest (テスト)

**設計判断:**
- 三項演算子 `?:` は Phase 1 では対応しない（`Case:` の `:` との競合リスク回避）
- `parse()` は `ParseResult = { file: FileNode; diagnostics: Diagnostic[] }` を返す（FileNode 自体に diagnostics を持たせない）
- ビルドは tsc のみ。esbuild は Phase 1 では使わない

```ts
// src/shared/keywords.ts
export const OBJECT_NAMES: readonly string[] = [
  "Axle", "Background", "Base", "Body", /* ... 現在の grammar から全て */ 
] as const;

export const CONTROL_KEYWORDS: readonly string[] = [
  "ApplySwitch", "Else", "If",
] as const;

export const CASE_KEYWORDS: readonly string[] = [
  "Case", "Default",
] as const;

export const PROPERTY_NAMES: readonly string[] = [
  "Acceleration", "AccelerationRel", "AirResistance", /* ... 現在の grammar から全て */
] as const;

export const CONSTANTS: readonly string[] = [
  "Add", "Alpha", "DayAlpha", "Down", "Env", /* ... */
  "no", "yes",
] as const;

export const OBJECT_NAME_SET = new Set(OBJECT_NAMES);
export const CONTROL_KEYWORD_SET = new Set(CONTROL_KEYWORDS);
export const CASE_KEYWORD_SET = new Set(CASE_KEYWORDS);
export const PROPERTY_NAME_SET = new Set(PROPERTY_NAMES);
export const CONSTANT_SET = new Set(CONSTANTS);

/** 識別子がいずれかのキーワードカテゴリに属するか */
export function classifyIdentifier(name: string): "object" | "control" | "case" | "property" | "constant" | "unknown" {
  if (OBJECT_NAME_SET.has(name)) return "object";
  if (CONTROL_KEYWORD_SET.has(name)) return "control";
  if (CASE_KEYWORD_SET.has(name)) return "case";
  if (PROPERTY_NAME_SET.has(name)) return "property";
  if (CONSTANT_SET.has(name)) return "constant";
  return "unknown";
}
```

**Files:**

- Create: `test/parser.test.ts`

**Step 1: テストを書く**

```ts
describe("parser", () => {
  // 基本
  it("should parse empty input → file with empty body, no diagnostics");
  it("should parse a simple object → ObjectNode with name, empty body");
  it("should parse object with string argument → args[0] is string ExprNode");
  it("should parse properties with comma-separated values");
  it("should parse nested objects");
  it("should parse comments (skipped in AST)");

  // If/Else — 具体的な AST 形状を検証
  it("should parse If/Else → IfNode with condition, then, else_", () => {
    const { file } = parse("Body { If 1 { Coord = 0; } Else { Coord = 1; } }");
    const body = file.body[0];
    expect(body.type).toBe("object");
    if (body.type === "object") {
      const ifNode = body.body[0];
      expect(ifNode.type).toBe("if");
      if (ifNode.type === "if") {
        expect(ifNode.condition.type).toBe("number");
        expect(ifNode.then).toHaveLength(1);
        expect(ifNode.else_).toHaveLength(1);
      }
    }
  });

  // ApplySwitch — 具体的な AST 形状を検証
  it("should parse ApplySwitch with Case/Default", () => {
    const { file } = parse('ApplySwitch "_FRONT" { Case 0: Coord = 0; Case 1: Coord = 1; Default: Coord = 2; }');
    const node = file.body[0];
    expect(node.type).toBe("applySwitch");
    if (node.type === "applySwitch") {
      expect(node.switchName.type).toBe("string");
      expect(node.cases).toHaveLength(2);
      expect(node.cases[0].values[0].type).toBe("number");
      expect(node.default_).toHaveLength(1);
    }
  });

  // 式の優先順位
  it("should respect operator precedence (1+2*3 = 1+(2*3))", () => {
    const { file } = parse("If 1+2*3 { }");
    if (file.body[0].type === "if") {
      const cond = file.body[0].condition;
      expect(cond.type).toBe("binary");
      if (cond.type === "binary") {
        expect(cond.op).toBe("+");
        expect(cond.right.type).toBe("binary");
        if (cond.right.type === "binary") {
          expect(cond.right.op).toBe("*");
        }
      }
    }
  });

  // エラーリカバリ
  it("should collect parse errors without crashing");
  it("should recover from errors at sync points and continue parsing");

  // 現実的な入力
  it("should parse a realistic RailSim2 file with no diagnostics");
});
```

**Step 2: テスト失敗確認**
**Step 3: parser.ts を実装**

再帰下降 + Pratt parser。`parse()` は `ParseResult` を返す。

**重要な設計判断: 未知識別子の扱い**
パーサーは構文レベルでのみ判定する。`identifier + {` は常に `ObjectNode` として、`identifier + =` は常に `PropertyNode` として構文的に受理する。識別子が既知キーワードかどうかの判定はパーサーの責務ではなく、後段の `unknownKeywordValidator` に委譲する。これにより:
- パーサーは `keywords.ts` に依存しない（純粋な構文解析）
- 未知キーワードが parse error にならず、AST に残る
- validator が warning として適切に報告できる

エラーリカバリ同期点: `;`, `}`, `Case`, `Default`, `Else`, `identifier + {` パターン, EOF。

describe("unknown keyword validator", () => {
  it("should not warn for known object names", () => {
    const { file } = parse("Body { Coord = 1; }");
    const diags = validateUnknownKeywords(file);
    expect(diags).toHaveLength(0);
  });

  it("should warn for unknown object names", () => {
    const { file } = parse("FooBar { }");
    const diags = validateUnknownKeywords(file);
    expect(diags).toHaveLength(1);
    expect(diags[0].severity).toBe("warning");
    expect(diags[0].message).toContain("FooBar");
  });

  it("should warn for unknown property names", () => {
    const { file } = parse("Body { UnknownProp = 1; }");
    const diags = validateUnknownKeywords(file);
    expect(diags).toHaveLength(1);
    expect(diags[0].message).toContain("UnknownProp");
  });

  it("should not warn for identifiers in value position", () => {
    // PluginType = Rail; — "Rail" is a constant, not a keyword in property position
    const { file } = parse("PluginHeader { PluginType = Rail; }");
    const diags = validateUnknownKeywords(file);
    expect(diags).toHaveLength(0);
  });
});
